# Sworcery Patcher

A modern, lightweight desktop application built with [Tauri](https://tauri.app/), Vanilla JavaScript, and Rust. 

This application serves as an interactive patcher designed to easily deploy Community Translation patches (such as Spanish and Portuguese) for the game **"Superbrothers: Sword & Sworcery EP"**.

## Features
- **Auto-Detection:** Automatically searches standard installation paths across multiple platforms to find your game folder.
- **Safe Patching:** Backs up your original `sworcery.dat` file as a `.bak` copy *before* applying the new language patch.
- **Rollback System:** Allows you to instantly remove the translation and restore your game to its vanilla state.
- **Fully Localized GUI:** The UI adapts to either Spanish (`es`) or Portuguese (`pt`) based on the environment variables.
- **Cinematic Theme:** Custom frameless window design with pixel-art aesthetics.

## Local Development Setup

To run this project locally, you will need to set up the general Tauri prerequisites for your operating system.

### 1. Prerequisites
- **Node.js** (v18 or higher recommended)
- **Rust & Cargo** (install via `rustup`)

#### OS-Specific Dependencies:
- **macOS:** Xcode Command Line Tools (`xcode-select --install`).
- **Linux (Ubuntu/Debian):** `sudo apt update && sudo apt install libwebkit2gtk-4.1-dev build-essential curl wget file libxdo-dev libssl-dev libayatana-appindicator3-dev librsvg2-dev`
- **Windows:** Microsoft C++ Build Tools (Installing Visual Studio with "Desktop development with C++" workload).

You can follow the official [Tauri Setup Guide](https://v2.tauri.app/start/prerequisites/) for more detailed, OS-specific instructions.

### 2. Installation
Clone the repository and install the Node frontend dependencies:

```bash
cd patcher
npm install
```

### 3. Running in Development Mode
The patcher relies on the translated `sworcery.dat` files generated by the root repository scripts. 

Before running the app, you must first copy your original, clean `sworcery.dat` file into the root of the repository (`../sworcery.dat`). Our internal scripts will use this base file to compile the translated versions for you.

You also need to set the decryption password used by the bash scripts. Make sure to export it in your terminal before running the app:

```bash
export SWORCERY_PASSWORD="your_password_here"
```

Once the original file is placed in the root and the password is set, use the built-in npm aliases below. These aliases will automatically trigger the underlying Bash scripts to generate the patched `.dat` files, copy them into the `src-tauri/assets` folder, and start the Vite + Tauri server with the correct UI language:

- **For the Spanish translation:**
  ```bash
  npm run dev:es
  ```

- **For the Portuguese translation:**
  ```bash
  npm run dev:pt
  ```

*(If you run on Windows and these aliases fail due to environment variable exportation syntax, you can install `cross-env` natively to support variable injection).*

### 4. Building Production Binaries
To compile a standalone executable / `.app` bundle for your OS:

```bash
npm run build:es
# or
npm run build:pt
```

The compiled binaries will be placed in `src-tauri/target/release/bundle/`.
